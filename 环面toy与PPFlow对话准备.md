## 一、你做的环面 toy model 到底是什么

### 1.1 环面（torus）是什么？

**甜甜圈**。数学上叫 T² = S¹ × S¹，就是两个圆的乘积。

你看一个甜甜圈表面上的任何一点，都有两种方式"转圈"：

- **θ（大圆角）**：绕甜甜圈的洞转一圈（从外面绕到里面再绕回来）
- **φ（小圆角）**：绕甜甜圈的管子本身转一圈（截面是个圆）

两个角都取值在 `[0, 2π)` —— 这是关键。

### 1.2 你的代码里是怎么实现的

`torus_experiment/torus_experiment.py` 前几行就能看到：

```python
def torus_parametric(theta, phi):
    x = (R_MAJOR + R_MINOR * torch.cos(phi)) * torch.cos(theta)
    y = (R_MAJOR + R_MINOR * torch.cos(phi)) * torch.sin(theta)
    z = R_MINOR * torch.sin(phi)
```

这是把两个角 `(θ, φ)` 映射到 3D 空间中一个甜甜圈表面上的点 `(x, y, z)`。

**约束**：所有真实数据严格躺在甜甜圈表面上。违反约束 = "点飞到甜甜圈外面了"。

### 1.3 你在环面上做的流匹配

1. 训练集：在甜甜圈表面均匀采 20000 个点
2. 训练一个速度场网络 $v_\theta(x, t)$
3. 采样时从噪声出发，沿速度场演化 50 步
4. **每步之后做投影** `project_to_torus(x)` —— 把飞出去的点拉回甜甜圈表面

投影操作：

```python
def project_to_torus(xyz):
    # 先找最近的大圆（R_MAJOR 半径的圆）上的点
    theta = atan2(y, x)
    # 然后在小圆周围找最近的点
    ...
```

**这是你整个 GENESIS 项目的最小 toy**：约束集是一个 2D 流形，投影可以闭式算，完全展示"投影流匹配"的数学骨架。

### 1.4 为什么选环面这个例子？

- **有解析形式**：投影能闭式算，容易验证对错
- **几何简单但非平凡**：不是球面（太简单），不是凸集（平面太简单）
- **带洞**：Betti 数 β₁=2，验证流匹配能生成有**拓扑结构**的样本
- **老师团队 PPFlow 就用这个** → 无缝对接

## 二、多肽的扭转角为什么落在环面上

### 2.1 多肽 / 蛋白质的几何基础

**多肽**就是一串氨基酸连成的小蛋白（通常 5-50 个氨基酸）。

每个氨基酸在多肽里的**骨架（backbone）**长这样：

```
...N─Cα─C(=O)─N─Cα─C(=O)─N─Cα─C(=O)...
   │  │   │    │  │   │    │  │   │
   H  R   O    H  R   O    H  R   O
```

- **N**（氮）、**Cα**（α 碳）、**C**（羰基碳）是骨架三个原子
- **R**（侧链）、**H**、**O** 挂在骨架上

### 2.2 骨架的构象由三个二面角决定

骨架的每个原子之间都有化学键。**键长和键角几乎是固定的**（被化学决定了）。但**键可以绕自己旋转** —— 这是自由度的来源。

每个氨基酸残基贡献 3 个**扭转角 / 二面角**（dihedral angles）：

| 角 | 绕哪根键转 | 取值 |
|---|---|---|
| **φ（phi）** | N─Cα 键 | [0, 2π) |
| **ψ（psi）** | Cα─C 键 | [0, 2π) |
| **ω（omega）** | C─N（肽键） | 几乎恒为 π（因为肽键有部分双键性质，刚性） |

由于 ω 基本是定值，**每个残基只贡献 2 个自由角（φ, ψ）**。

### 2.3 扭转角为什么落在圆上（S¹）

**物理直觉**：扭转 360° 回到原样。扭 1° 和扭 361° 是**完全同一个构象**。

**数学本质**：每个扭转角是 SO(2) 群的参数，即 1 维旋转群。它的参数空间是：

$$
S^1 = \mathbb{R} / 2\pi\mathbb{Z}
$$

也就是**把实数轴上每隔 2π 就粘到一起** —— 得到一个圆。

**这不是约定而已，是几何的硬事实**：

- 如果把 φ 当成 ℝ 里的数，模型会把 0° 和 360° 当成"差了 360 个单位"的极远点
- 实际上它们**是同一个物理构象**
- 所以你做 FM 时必须显式处理这个"圆"结构，否则在 0/2π 边界附近会出现荒唐的预测

### 2.4 N 个残基 → 高维环面

N 个残基的多肽，有 2N 个独立扭转角（φ₁, ψ₁, φ₂, ψ₂, ..., φ_N, ψ_N）。

**每个 φᵢ 活在 S¹ 上，每个 ψᵢ 也活在 S¹ 上**。整个构象空间就是这些圆的笛卡尔乘积：

$$
T^{2N} = \underbrace{S^1 \times S^1 \times \cdots \times S^1}_{2N \text{ 个}}
$$

一个 10 残基的多肽 → 20 维环面 T²⁰。

你在做的就是 **T²** 的情况（2 个角），PPFlow 做的是 **T²ᴺ** 的情况（同样的几何结构，高维版）。

### 2.5 Ramachandran plot — 扭转角上不是均匀分布

扭转角的**取值范围**是 T²ᴺ，但不是所有点都是**物理上合理**的。

经典的 **Ramachandran plot** 把每个氨基酸的 (φ, ψ) 画在 2D 环面上：

```
        ψ (0→2π)
        ▲
2π ┤ ████░░░░░░░░░░
   │ ████░░░░░░░░░░   ← β-sheet 区
   │ ████░░░░░░░░░░
   │ ░░░░░░░░░░░░░░
π  ┤ ░░░░░░░░░░░░░░
   │ ░░░░████████░░   ← α-helix 区
   │ ░░░░████████░░
0  ┤ ░░░░░░░░░░░░░░
   └──────────────→ φ (0→2π)
```

**"密度集中"**的区域是能量最低的构象（α-螺旋、β-折叠）。其它区域因为原子碰撞等原因，物理上基本不出现。

所以**多肽构象的真实数据分布是 T²ᴺ 上一个非平凡的分布**，有密有疏。

## 三、两件事的连接（你准备跟李老师讲的）

### 3.1 数学结构一模一样

|  | 你的 toy | PPFlow |
|---|---|---|
| 流形 | T² 嵌入 R³（甜甜圈表面） | T²ᴺ 内在（N 残基扭转角） |
| 维度 | 2 | 2N |
| 数据 | 均匀 or 结构化分布 | Ramachandran 型非均匀分布 |
| 约束 | 点必须在 T² 表面 | 角必须在 [0, 2π) 内，整体在 T²ᴺ 上 |
| 投影 | `project_to_torus` 找最近点 | 对每个角做 `angle mod 2π`（更简单因为是内在坐标） |
| 流匹配 | 速度场定义在 R³，投影拉回 T² | 速度场直接定义在 T²ᴺ 上（内在版） |

**投影等价于"每步模 2π"**：

- 你的 3D 嵌入版：投影 = "找最近的甜甜圈表面点"
- PPFlow 内在版：投影 = "把角取到 [0, 2π)"
- 两者拓扑上完全一样

### 3.2 面对老师的对话剧本

**你开口**：

> "老师，我想问您一个具体的问题。我跑过环面上的投影流匹配 toy —— 就是在 T² 嵌入 3D 里做的 GENESIS 最小验证。您团队 PPFlow 也用了 torus，但看起来是把扭转角作为内在坐标直接在 T²ᴺ 上做流匹配。**能不能给我讲讲多肽的 (φ, ψ) 扭转角为什么天然落在环面上？**"

**如果他反问"你怎么理解"**，你的标准答案：

> "每个扭转角 φ 或 ψ 都是 1D 旋转群 SO(2) 的参数，几何上是 S¹。0° 和 360° 是同一物理构象。N 个残基 × 2 个角 / 残基，整个构象空间就是 T²ᴺ。我做的 toy 是 N=1 的特例（2D 环面）。"

**他大概会接着讲**（准备听这些）：

1. **Ramachandran 分布**：构象不是均匀的，α-螺旋、β-折叠是密集区
2. **为什么 FM 比 diffusion 好**：因为 torus 有几何对称性（循环群），FM 直接在 T²ᴺ 上定义，diffusion 的高斯加噪破坏循环结构
3. **PPFlow 的具体创新**：可能是某种 torus-specific 的 prior path 或者约束施加
4. **ω 为什么不算**：肽键的部分双键性质让 ω 几乎固定

## 四、30 秒自我测试（能答这三个就够了）

1. **"T² 是什么"** → 两个圆的乘积，甜甜圈表面
2. **"多肽为什么是 T²ᴺ"** → 每残基 2 个扭转角（φ, ψ），每个角活在 S¹ 上
